# AI CV Studio UI Prompt 计划

在实现或修改前端 UI 之前，先使用这份提示词。它不是审美建议，而是前端可还原的 UI 规格：版式、层级、组件状态、双语排版和品牌 Logo 都要按这里执行。

## 角色

你正在为 AI CV Studio 构建前端界面。它是一个中英双语 AI 简历编辑器，面向真实求职者，不是营销展示页。首屏必须像一个已经能工作的专业工具，而不是介绍产品概念的 landing page。

设计师人格：高级、克制、有强迫症。每一个像素、间距、字号、边框、对齐关系都必须有理由。界面可以有个性，但不能牺牲可读性和可信度。

## 产品目标

UI 应该帮助用户完成这些事情：

- 上传旧简历，或粘贴简历文本开始编辑。
- 把简历渲染成接近最终 PDF 的可预览成品。
- 点击分析按钮，在右侧获得 AI 优化建议。
- 逐条应用、忽略或编辑 AI 优化建议。
- 在左侧直接编辑简历。
- 输入新增经历，让 AI 自动放入简历合适位置。
- 支持照片、完整基本信息和多页简历预览。
- 让用户清楚看到哪些改动需要确认，哪些改动可以直接进入成品。

核心原则必须在界面中被看见：

> 只基于用户提供的信息改简历，不替用户编造事实。

## V1 范围约束

当前 V1 面向自用和朋友试用，先做可信的 AI 简历编辑器。

V1 不把求职信、面试自我介绍、多版本管理或公开 SaaS 账号体系作为主流程。除非产品 owner 明确要求，否则 UI 优先服务这个闭环：

上传旧简历或粘贴文本 -> AI 分析 -> 右侧建议 -> 左侧编辑 -> 新增经历自动插入 -> 导出 PDF。

AI 分析结果必须来自真实后端 AI provider 调用。UI 不能把本地 mock 或固定规则建议展示成“AI 已分析成功”。

## 整体设计方向

### 关键词

- 专业工作台
- 简历编辑器
- 成品预览
- 菠萝品牌记忆点
- 中英双语稳定排版
- 高密度但不混乱

### 视觉概念

使用「Pineapple Studio / 菠萝简历工坊」作为视觉概念。

菠萝不是可爱的水果插画，而是一个品牌隐喻：

- 菠萝外壳的菱格纹 = 简历结构、内容层级、可编辑章节。
- 菠萝叶冠 = 生长、职业路径、向上延展。
- 菠萝金色果肉 = 真实经历里提炼出的成果。
- 深绿色外壳 = 可信、克制、专业。

界面应该像一个高级简历编辑室：有严谨的网格、有清楚的编辑状态、有接近最终 PDF 的预览，同时带一点菠萝品牌的锐利个性。

### 必须避免

- 不要做营销页英雄区。
- 不要使用通用紫色 SaaS 渐变。
- 不要使用装饰性渐变光球、bokeh、漂浮 orb。
- 不要用大面积单一米色、奶油色、棕橙色做成廉价复古风。
- 不要把所有模块都做成圆角大卡片。
- 不要套娃式卡片嵌套卡片。
- 不要承诺“保证通过 ATS”“保证拿到面试”。

## 品牌 Logo 设计

### Logo 名称

品牌标记建议命名为 `Pineapple Studio Mark`。

### Logo 结构

Logo 由 3 个元素组成：

1. 菠萝主体：竖向六边形/圆角菱形轮廓，不能是普通椭圆水果。
2. 结构网格：主体内部使用 5-7 条斜线交叉形成菱格，每个交点像一个 section node。
3. 叶冠：顶部 3 片几何叶片，像简历页签/书签，也像向上的职业路径。

### Logo 性格

Logo 要高级、个性、可识别。不能可爱卡通，不能像果汁店，不能像旅游品牌。

推荐方向：

- 线条硬朗，局部圆角。
- 主体接近印章或徽章，有“可信认证”的感觉。
- 叶冠可以略微不对称，让它有记忆点。
- 内部菱格必须与产品的结构化简历编辑产生关联。

### Logo 尺寸规格

- 基础画布：`64 x 64`
- 安全边距：`6px`
- 主体轮廓：从 `x=18 y=20` 到 `x=46 y=56`
- 叶冠高度：`8px - 20px`
- 最小可用尺寸：`24 x 24`
- favicon 尺寸下只保留轮廓、2 条菱格线、3 片叶冠，不显示复杂节点。

### Logo 配色

使用以下品牌 token，不要随意新增相近颜色：

```css
:root {
  --brand-ink: #122421;
  --brand-pine: #1f4b3f;
  --brand-leaf: #3f7a50;
  --brand-gold: #f0b83a;
  --brand-gold-deep: #c9821d;
  --brand-ivory: #fff8e6;
  --brand-paper: #f6f1e7;
  --brand-coral: #dd5f3a;
  --brand-blue: #2f5f8f;
}
```

主色比例：

- 45% 深墨绿 `--brand-ink / --brand-pine`
- 25% 象牙白和纸张底色
- 18% 菠萝金
- 7% 叶绿色
- 5% 珊瑚橙或深蓝作为状态强调

### Logo SVG 还原提示

前端实现 Logo 时优先使用 inline SVG，方便控制颜色和响应式尺寸。不要用 emoji、PNG 截图或纯文字替代。

SVG 应符合这个结构：

```tsx
function PineappleLogo({ size = 40 }: { size?: number }) {
  return (
    <svg width={size} height={size} viewBox="0 0 64 64" aria-label="AI CV Studio">
      <path className="leaf leaf-left" d="M31 19 L22 9 L29 12 Z" />
      <path className="leaf leaf-mid" d="M33 20 L34 6 L39 16 Z" />
      <path className="leaf leaf-right" d="M36 20 L48 11 L43 22 Z" />
      <path className="body" d="M32 18 C42 20 50 31 47 43 C44 55 36 59 27 56 C18 53 14 43 17 32 C19 24 24 19 32 18 Z" />
      <path className="grid" d="M20 31 L40 55 M17 39 L30 56 M27 20 L47 42 M38 23 L18 43 M45 32 L25 56" />
      <circle className="node" cx="31" cy="38" r="1.8" />
      <circle className="node" cx="37" cy="45" r="1.8" />
    </svg>
  )
}
```

样式要求：

- `.body` 使用 `--brand-gold` 填充、`--brand-ink` 描边。
- `.grid` 使用 `--brand-ink`，线宽 `2`，透明度 `0.72`。
- `.leaf` 使用 `--brand-leaf` 填充、`--brand-ink` 描边。
- `.node` 使用 `--brand-ink` 或 `--brand-coral`，只在大于 `32px` 时显示。
- 轮廓边角可以圆润，但不要超过 `8px` 的视觉圆角。

### Logo 应用

- 顶部品牌区：Logo `40px`，右侧两行文字：`AI CV Studio` + `AI-assisted resumes`。
- 移动端顶部：Logo `36px`，品牌文字只保留 `AI CV`。
- 空状态：可以放大成 `72px` 的半透明菠萝简历印章。
- 加载状态：不旋转整个菠萝；只让内部结构节点依次点亮。

## 版式总览

### 桌面布局

目标桌面宽度：`1440px`。

使用两层主结构：

1. 中央工作区：最小 `640px`，承载简历编辑和预览。
2. 右侧建议区：`360px - 460px`，承载 AI 优化建议和新增经历插入结果。

顶部工具栏承载品牌、上传、粘贴、分析和导出。不要为了产品看起来复杂而增加独立左侧导航。

推荐 CSS：

```css
.app-shell {
  display: grid;
  grid-template-columns: minmax(640px, 1fr) minmax(360px, 440px);
  min-height: 100svh;
}
```

如果当前代码需要保留窄侧栏，只能用于品牌和少量状态信息，不能把 `Job`、`Questions` 等旧流程导航放回 V1 主界面。

### 首屏信息架构

首屏必须一眼看见完整工作流：

- 顶部：品牌、上传旧简历、粘贴文本、分析按钮、导出按钮。
- 左侧或中间主体：可编辑简历和接近最终 PDF 的多页预览。
- 右侧：AI 优化建议和新增经历入口。

首屏标题不使用营销话术。推荐文案：

- 中文：`上传或粘贴简历，让 AI 帮你找出可改进的地方。`
- 英文：`Paste or upload a resume, then edit with AI suggestions.`

副标题最多 2 行，不超过 `88` 个中文字符或 `150` 个英文字符。

### 移动端布局

390px 宽度下使用分步骤布局，不强行塞三栏：

1. `Import`
2. `Resume`
3. `Suggest`
4. `Add`
5. `Export`

移动端底部使用 sticky step nav，高度 `56px`。每个 tab 使用图标 + 5 字以内标签，不要长文字。

移动端模块宽度：

- 页面 padding：`14px`
- 模块间距：`12px`
- 面板 padding：`16px`
- 按钮最小高度：`44px`
- 输入区最小高度：`120px`

## 栅格、间距和圆角

### 间距系统

只使用这些 spacing token：

```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 20px;
--space-6: 24px;
--space-8: 32px;
--space-10: 40px;
--space-12: 48px;
```

规则：

- 面板内边距桌面 `24px`，移动端 `16px`。
- 相邻面板间距桌面 `16px - 20px`，移动端 `12px`。
- 标题和正文间距 `8px - 12px`。
- 操作按钮区距离内容 `20px - 24px`。

### 圆角

- 主工作台面板：`8px`
- 输入框、按钮、标签：`6px`
- Logo 和图标容器：`8px` 或硬边 `0px`
- 简历预览纸张：`4px`

不要使用 `20px+` 的大圆角，产品不是玩具感 SaaS。

### 边框和阴影

边框优先于阴影。阴影只用于强调层级，不能让界面变成漂浮卡片。

```css
--border-subtle: 1px solid rgba(18, 36, 33, 0.12);
--border-strong: 1px solid rgba(18, 36, 33, 0.28);
--shadow-panel: 0 18px 48px rgba(18, 36, 33, 0.08);
--shadow-raised: 0 24px 70px rgba(18, 36, 33, 0.14);
```

## 排版系统

### 字体

优先使用系统里稳定可用但不廉价的组合：

```css
--font-sans: "Avenir Next", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif;
--font-serif: "Songti SC", "STSong", Georgia, serif;
--font-mono: "SFMono-Regular", "Cascadia Mono", Consolas, monospace;
```

使用规则：

- 产品界面：`--font-sans`
- 简历预览姓名、章节标题：可以少量使用 `--font-serif`，制造“文档感”
- 数据、来源 ID、百分比、文件格式：`--font-mono`

### 字号层级

桌面：

- H1：`56px / 0.98`
- H2：`28px / 1.15`
- H3：`20px / 1.25`
- Body：`15px - 16px / 1.6`
- Small：`12px - 13px / 1.4`
- Mono label：`11px - 12px / 1.2`

移动端：

- H1：`34px / 1.04`
- H2：`23px / 1.15`
- H3：`18px / 1.25`
- Body：`15px / 1.58`

禁止使用 viewport 宽度动态缩放字体。可以用 `clamp()` 限制范围，但不要让文字随着窗口连续跳动。

### 中英文排版

- 中文按钮最多 8 个汉字，英文按钮最多 18 个字符。
- 英文长词必须允许换行：`overflow-wrap: anywhere;`
- 所有标题容器必须允许两行高度。
- 中英文切换时组件高度可以增长，但不能横向溢出。
- 文案不允许被 `text-overflow: ellipsis` 截掉，除非是文件名、来源 ID 或日志片段。

## 核心页面模块

### 1. 顶部工具栏

顶部工具栏不是营销 hero，而是编辑器工具栏。

必须包含：

- Pineapple Studio Mark Logo
- `AI CV Studio`
- 保存状态，例如 `已保存草稿`
- 上传旧简历按钮
- 粘贴文本入口
- 分析按钮
- 导出 PDF 按钮

视觉：

- 高度 `64px - 72px`
- 背景用纸张色或深墨绿，两者选一个，不要半透明玻璃风。
- 主操作是 `分析简历`，导出按钮次之。
- 移动端可以把上传、分析、导出放进底部 sticky 操作栏。

### 2. 简历编辑区

简历编辑区是主视图，不能被建议面板压缩得过小。

必须包含：

- 可编辑简历正文。
- 接近 A4 的预览感。
- 多页页面容器和页间距。
- 照片上传、照片预览和基本裁剪状态。
- 姓名、电话、邮箱、城市、求职意向等基本信息。
- 基本信息、教育、工作、项目、技能等区块。
- 字数或保存状态。

布局：

- 桌面：左侧或中间，占页面主要宽度。
- 移动：优先展示编辑区，建议面板可切换查看。
- 文本输入必须支持长简历，不出现横向滚动。
- 内容超过一页时显示第 2 页、第 3 页，而不是继续压缩第一页。
- 页面分隔要清楚，让用户知道 PDF 会导出多页。

### 3. 上传和粘贴入口

上传区不是巨大空白拖拽框，而是紧凑的导入面板。

必须包含：

- 文件上传入口：PDF / DOCX / TXT / MD
- 粘贴简历文本入口
- 隐私提示：`文件仅用于分析和编辑你的简历，不会展示给其他人。`

状态：

- 空：展示上传和粘贴两个入口。
- 拖拽中：边框变菠萝金，背景轻微变亮。
- 解析中：显示文件名、进度和当前步骤。
- 成功：进入可编辑简历草稿。
- 失败：明确原因和下一步，例如 `PDF 无法提取文本，请上传 DOCX 或粘贴文本。`

### 4. 右侧 AI 建议

右侧建议区是辅助面板，不是主结果。

每条建议必须包含：

- 建议标题。
- 对应简历位置。
- 问题说明。
- 推荐改法。
- 可选示例改写。
- 风险或需要确认的事实。

建议操作：

- 应用。
- 忽略。
- 复制。
- 标记已处理。

每条建议的 `exampleRewrite` 必须像可直接放进简历的正文。UI 如果检测到明显占位提示语，例如 `请补充`、`根据实际情况`、`你确认过`、`结果或范围`，应阻断一键应用并提示这是不可应用建议。

如果 AI 没有建议，显示明确空状态，例如 `这版简历暂时没有明显结构问题，可以继续手动编辑或导出。`

### 5. 新增经历入口

新增经历入口用于把用户的新内容自动放入简历。

必须包含：

- 公司名输入。
- 职位输入。
- 入职开始时间。
- 离职结束时间或“至今”。
- 主要项目名称。
- 项目介绍。
- 主要职责、关键动作和结果输入。
- 新增经历 textarea，用于补充自由描述。
- `插入到简历` 按钮。
- 插入位置说明，例如 `建议放入：项目经历 / 外呼运营管理系统`。
- 插入后的预览。
- 接受、编辑、撤销操作。

用户输入示例：

```text
新增经历：我做了 Redis 缓存优化，把接口平均响应时间从 800ms 降到 300ms。
```

如果 AI 无法判断插入位置，必须让用户选择章节，不要乱插。

AI 不能替用户补写公司、职位、时间、项目结果或薪资信息。缺失信息应以表单校验、提示问题或待确认状态呈现。

### 6. 导出区

V1 优先：

- PDF
- Copy

DOCX 可以作为 V1.1。除非产品 owner 明确要求，不要把 DOCX 做成首要按钮。

导出前必须有校验：

- 是否有未确认的 AI 插入内容。
- 是否有明显空白关键字段。
- 当前预览是否可生成 PDF。
- 多页内容是否完整进入导出范围。
- 照片是否加载成功且比例正常。

## 组件规范

### 按钮

按钮分三种：

- Primary：深墨绿底，象牙白字，用于下一步主操作。
- Secondary：纸张底，深墨绿边框，用于上传、复制、切换。
- Danger/Block：珊瑚红边框或底色，用于无法应用或高风险建议。

按钮必须有：

- hover
- active
- focus-visible
- disabled
- loading

Focus 样式：

```css
button:focus-visible,
a:focus-visible,
input:focus-visible,
textarea:focus-visible {
  outline: 3px solid rgba(240, 184, 58, 0.55);
  outline-offset: 2px;
}
```

### 输入框

- 文本输入和 textarea 使用 `6px` 圆角。
- 边框默认 `rgba(18, 36, 33, 0.18)`。
- focus 时边框变 `--brand-gold-deep`。
- 错误状态显示红色边框 + 明确错误文案。
- textarea 右下角显示字数或解析状态。

### 标签

标签要服务信息识别，不要装饰。

推荐标签类型：

- 文件格式：`PDF`、`DOCX`
- 来源：`Upload`、`Paste`、`New experience`
- 状态：`Applied`、`Needs confirmation`、`Skipped`
- 语言：`ZH`、`EN`

### 图标

如果使用 icon library，优先使用 lucide icon。不要手写通用图标，Logo 除外。

图标按钮必须有 tooltip 或 `aria-label`。工具类操作优先用图标，例如复制、下载、删除、展开、收起。

## 动效

动效必须低调、可信、服务反馈。

允许：

- 页面进入时面板 `opacity + translateY(8px)`，总时长 `180ms - 240ms`。
- 建议被应用时轻微点亮。
- 上传进度条线性推进。
- Logo 大尺寸加载时内部节点依次亮起。

禁止：

- 大面积弹跳。
- 无限旋转 Logo。
- 背景粒子雨。
- 让简历预览漂浮摇晃。

## 必须考虑的 UI 状态

每个新流程都要考虑：

- 空状态
- 加载状态
- 成功状态
- 校验错误
- 上传失败
- AI 分析失败
- 新增经历无法判断插入位置
- 未确认内容阻断导出
- 多页内容导出完整性
- 照片加载失败或比例异常
- 移动端布局
- 英文文案变长后的布局
- 键盘 focus
- 低网速或接口超时

## 页面文案原则

文案要像专业产品，不要像营销广告。

推荐：

- `建议补充项目结果，否则这段说服力偏弱。`
- `这条新增经历建议放入项目经历。`
- `PDF 无法提取文本，请改用粘贴。`
- `导出前请确认 AI 插入的新增经历。`

避免：

- `一键打造完美简历`
- `保证 ATS 通过`
- `秒杀 HR`
- `AI 自动帮你搞定所有求职材料`

## 验收标准

视觉验收：

- 1440px 桌面宽度下，首屏能同时看见简历编辑区、右侧建议、上传/分析/导出入口。
- 390px 移动端宽度下没有横向溢出。
- 中文和英文界面文案都不能被裁切。
- 面板之间的对齐、间距、边框粗细统一。
- Logo 在 `24px`、`40px`、`72px` 三种尺寸下都清晰。
- 页面不能读成普通紫色 SaaS 模板，也不能读成果汁/水果品牌。

交互验收：

- 按钮动作明确，并且有可见的 focus 状态。
- 上传、解析、AI 分析、新增经历插入、导出失败都有明确状态。
- 无法确认的新增内容不能无提示进入最终简历。
- AI 建议可应用、可忽略，应用后预览立即变化。
- 多页预览与 PDF 导出页数一致。
- 移动端所有主流程都可以单手完成。

产品验收：

- 首屏能表达产品工作流程，但不是纯营销页。
- 用户能上传旧简历或粘贴文本。
- 用户能以结构化表单输入新增工作经历并看到插入位置。
- 用户能上传或编辑照片，并在预览和导出中看到一致效果。
- 不承诺不现实的自动化结果。
- 用户能明确知道下一步该做什么。
