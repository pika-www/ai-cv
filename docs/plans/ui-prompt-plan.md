# AI CV Studio UI Prompt 计划

在实现或修改前端 UI 之前，先使用这份提示词。它不是审美建议，而是前端可还原的 UI 规格：版式、层级、组件状态、双语排版和品牌 Logo 都要按这里执行。

## 角色

你正在为 AI CV Studio 构建前端界面。它是一个中英双语 AI 求职材料工作台，面向真实求职者，不是营销展示页。首屏必须像一个已经能工作的专业工具，而不是介绍产品概念的 landing page。

设计师人格：高级、克制、有强迫症。每一个像素、间距、字号、边框、对齐关系都必须有理由。界面可以有个性，但不能牺牲可读性和可信度。

## 产品目标

UI 应该帮助用户完成这些事情：

- 上传已有简历，或从空白问答开始创建职业档案。
- 建立可验证的真实经历库。
- 粘贴目标岗位 JD，并理解匹配度和缺口。
- 回答 AI 追问，从真实经历里挖出成果、职责、数据和证据。
- 预览针对目标 JD 的简历 bullet 和来源追踪。
- 看清楚每条生成内容是否有来源依据。

核心原则必须在界面中被看见：

> 没有证据，不写进简历。

## V1 范围约束

当前 V1 面向自用和朋友试用，先做可信的 JD 定制简历生成器。

V1 不把求职信、面试自我介绍、多版本管理或公开 SaaS 账号体系作为主流程。除非产品 owner 明确要求，否则 UI 优先服务这个闭环：

旧简历/经历 + 目标 JD -> 证据库 -> AI 追问 -> 定制简历 bullet -> 来源追踪 -> 复制或导出。

## 整体设计方向

### 关键词

- 专业工作台
- 证据驱动
- 简历工坊
- 菠萝品牌记忆点
- 中英双语稳定排版
- 高密度但不混乱

### 视觉概念

使用「Pineapple Ledger / 菠萝证据账本」作为视觉概念。

菠萝不是可爱的水果插画，而是一个品牌隐喻：

- 菠萝外壳的菱格纹 = 证据网格、来源追踪、结构化经历。
- 菠萝叶冠 = 生长、职业路径、向上延展。
- 菠萝金色果肉 = 真实经历里提炼出的成果。
- 深绿色外壳 = 可信、克制、专业。

界面应该像一个高级资料编辑室：有严谨的网格、有清楚的证据标签、有可操作的产出预览，同时带一点菠萝品牌的锐利个性。

### 必须避免

- 不要做营销页英雄区。
- 不要使用通用紫色 SaaS 渐变。
- 不要使用装饰性渐变光球、bokeh、漂浮 orb。
- 不要用大面积单一米色、奶油色、棕橙色做成廉价复古风。
- 不要把所有模块都做成圆角大卡片。
- 不要套娃式卡片嵌套卡片。
- 不要承诺“保证通过 ATS”“保证拿到面试”。

## 品牌 Logo 设计

### Logo 名称

品牌标记建议命名为 `Pineapple Proof Mark`。

### Logo 结构

Logo 由 3 个元素组成：

1. 菠萝主体：竖向六边形/圆角菱形轮廓，不能是普通椭圆水果。
2. 证据网格：主体内部使用 5-7 条斜线交叉形成菱格，每个交点像一个 evidence node。
3. 叶冠：顶部 3 片几何叶片，像简历页签/书签，也像向上的职业路径。

### Logo 性格

Logo 要高级、个性、可识别。不能可爱卡通，不能像果汁店，不能像旅游品牌。

推荐方向：

- 线条硬朗，局部圆角。
- 主体接近印章或徽章，有“可信认证”的感觉。
- 叶冠可以略微不对称，让它有记忆点。
- 内部菱格必须与产品的证据追踪产生关联。

### Logo 尺寸规格

- 基础画布：`64 x 64`
- 安全边距：`6px`
- 主体轮廓：从 `x=18 y=20` 到 `x=46 y=56`
- 叶冠高度：`8px - 20px`
- 最小可用尺寸：`24 x 24`
- favicon 尺寸下只保留轮廓、2 条菱格线、3 片叶冠，不显示复杂节点。

### Logo 配色

使用以下品牌 token，不要随意新增相近颜色：

```css
:root {
  --brand-ink: #122421;
  --brand-pine: #1f4b3f;
  --brand-leaf: #3f7a50;
  --brand-gold: #f0b83a;
  --brand-gold-deep: #c9821d;
  --brand-ivory: #fff8e6;
  --brand-paper: #f6f1e7;
  --brand-coral: #dd5f3a;
  --brand-blue: #2f5f8f;
}
```

主色比例：

- 45% 深墨绿 `--brand-ink / --brand-pine`
- 25% 象牙白和纸张底色
- 18% 菠萝金
- 7% 叶绿色
- 5% 珊瑚橙或深蓝作为状态强调

### Logo SVG 还原提示

前端实现 Logo 时优先使用 inline SVG，方便控制颜色和响应式尺寸。不要用 emoji、PNG 截图或纯文字替代。

SVG 应符合这个结构：

```tsx
function PineappleLogo({ size = 40 }: { size?: number }) {
  return (
    <svg width={size} height={size} viewBox="0 0 64 64" aria-label="AI CV Studio">
      <path className="leaf leaf-left" d="M31 19 L22 9 L29 12 Z" />
      <path className="leaf leaf-mid" d="M33 20 L34 6 L39 16 Z" />
      <path className="leaf leaf-right" d="M36 20 L48 11 L43 22 Z" />
      <path className="body" d="M32 18 C42 20 50 31 47 43 C44 55 36 59 27 56 C18 53 14 43 17 32 C19 24 24 19 32 18 Z" />
      <path className="grid" d="M20 31 L40 55 M17 39 L30 56 M27 20 L47 42 M38 23 L18 43 M45 32 L25 56" />
      <circle className="node" cx="31" cy="38" r="1.8" />
      <circle className="node" cx="37" cy="45" r="1.8" />
    </svg>
  )
}
```

样式要求：

- `.body` 使用 `--brand-gold` 填充、`--brand-ink` 描边。
- `.grid` 使用 `--brand-ink`，线宽 `2`，透明度 `0.72`。
- `.leaf` 使用 `--brand-leaf` 填充、`--brand-ink` 描边。
- `.node` 使用 `--brand-ink` 或 `--brand-coral`，只在大于 `32px` 时显示。
- 轮廓边角可以圆润，但不要超过 `8px` 的视觉圆角。

### Logo 应用

- 侧边栏品牌区：Logo `44px`，右侧两行文字：`AI CV Studio` + `Evidence-first resumes`。
- 移动端顶部：Logo `36px`，品牌文字只保留 `AI CV`。
- 空状态：可以放大成 `72px` 的半透明菠萝证据印章。
- 加载状态：不旋转整个菠萝；只让内部证据节点依次点亮。

## 版式总览

### 桌面布局

目标桌面宽度：`1440px`。

使用三层结构：

1. 左侧导航：固定宽度 `280px`。
2. 中央工作区：最小 `520px`，承载上传、JD、追问。
3. 右侧结果区：`360px - 460px`，承载简历预览、来源追踪和导出。

推荐 CSS：

```css
.app-shell {
  display: grid;
  grid-template-columns: 280px minmax(520px, 1fr) minmax(360px, 440px);
  min-height: 100svh;
}
```

如果当前代码仍是两栏布局，至少要把右侧结果预览做成视觉上的固定结果区域，不要让所有模块无差别堆叠。

### 首屏信息架构

首屏必须一眼看见完整工作流：

- 左侧：品牌、导航、产品原则。
- 中间上方：当前任务状态和下一步动作。
- 中间主体：上传/粘贴经历、JD 分析、AI 追问。
- 右侧：生成结果预览和证据状态。

首屏标题不使用营销话术。推荐文案：

- 中文：`基于真实经历，生成可追溯的岗位版简历。`
- 英文：`Build a traceable resume from verified career evidence.`

副标题最多 2 行，不超过 `88` 个中文字符或 `150` 个英文字符。

### 移动端布局

390px 宽度下使用分步骤布局，不强行塞三栏：

1. `Profile`
2. `Job`
3. `Questions`
4. `Resume`
5. `Export`

移动端底部使用 sticky step nav，高度 `56px`。每个 tab 使用图标 + 5 字以内标签，不要长文字。

移动端模块宽度：

- 页面 padding：`14px`
- 模块间距：`12px`
- 面板 padding：`16px`
- 按钮最小高度：`44px`
- 输入区最小高度：`120px`

## 栅格、间距和圆角

### 间距系统

只使用这些 spacing token：

```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 20px;
--space-6: 24px;
--space-8: 32px;
--space-10: 40px;
--space-12: 48px;
```

规则：

- 面板内边距桌面 `24px`，移动端 `16px`。
- 相邻面板间距桌面 `16px - 20px`，移动端 `12px`。
- 标题和正文间距 `8px - 12px`。
- 操作按钮区距离内容 `20px - 24px`。

### 圆角

- 主工作台面板：`8px`
- 输入框、按钮、标签：`6px`
- Logo 和图标容器：`8px` 或硬边 `0px`
- 简历预览纸张：`4px`

不要使用 `20px+` 的大圆角，产品不是玩具感 SaaS。

### 边框和阴影

边框优先于阴影。阴影只用于强调层级，不能让界面变成漂浮卡片。

```css
--border-subtle: 1px solid rgba(18, 36, 33, 0.12);
--border-strong: 1px solid rgba(18, 36, 33, 0.28);
--shadow-panel: 0 18px 48px rgba(18, 36, 33, 0.08);
--shadow-raised: 0 24px 70px rgba(18, 36, 33, 0.14);
```

## 排版系统

### 字体

优先使用系统里稳定可用但不廉价的组合：

```css
--font-sans: "Avenir Next", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif;
--font-serif: "Songti SC", "STSong", Georgia, serif;
--font-mono: "SFMono-Regular", "Cascadia Mono", Consolas, monospace;
```

使用规则：

- 产品界面：`--font-sans`
- 简历预览姓名、章节标题：可以少量使用 `--font-serif`，制造“文档感”
- 数据、来源 ID、百分比、文件格式：`--font-mono`

### 字号层级

桌面：

- H1：`56px / 0.98`
- H2：`28px / 1.15`
- H3：`20px / 1.25`
- Body：`15px - 16px / 1.6`
- Small：`12px - 13px / 1.4`
- Mono label：`11px - 12px / 1.2`

移动端：

- H1：`34px / 1.04`
- H2：`23px / 1.15`
- H3：`18px / 1.25`
- Body：`15px / 1.58`

禁止使用 viewport 宽度动态缩放字体。可以用 `clamp()` 限制范围，但不要让文字随着窗口连续跳动。

### 中英文排版

- 中文按钮最多 8 个汉字，英文按钮最多 18 个字符。
- 英文长词必须允许换行：`overflow-wrap: anywhere;`
- 所有标题容器必须允许两行高度。
- 中英文切换时组件高度可以增长，但不能横向溢出。
- 文案不允许被 `text-overflow: ellipsis` 截掉，除非是文件名、来源 ID 或日志片段。

## 核心页面模块

### 1. 左侧导航

内容：

- Pineapple Proof Mark Logo
- `AI CV Studio`
- 小副标题：`Evidence-first resumes`
- 导航项：`Workspace`、`Evidence`、`JD Match`、`Questions`、`Resume`
- 底部产品原则：`No evidence, no resume claim.`

视觉：

- 宽度 `280px`
- 背景深墨绿或接近白的纸张色，两者选一个，不要半透明玻璃风。
- 当前导航项左侧使用 `4px` 菠萝金竖线。
- 导航数字用 mono 字体，例如 `01`。

### 2. 任务状态区

放在工作区顶部，不要做成营销 hero。

必须展示：

- 当前流程阶段，例如 `Step 2 / JD Match`
- 证据完整度，例如 `Evidence readiness 68%`
- 下一步建议，例如 `补充 2 条含数字的项目结果`
- 主操作按钮，例如 `Analyze JD`

布局：

- 桌面：横向一行，左文右按钮。
- 移动：纵向堆叠，按钮满宽。

### 3. 上传和经历输入

上传区不是巨大空白拖拽框，而是紧凑的 intake panel。

必须包含：

- 文件上传入口：PDF / DOCX / TXT
- 粘贴简历入口
- 手动新增经历入口
- 隐私提示：`文件仅用于生成你的材料，不会被展示给其他人。`

状态：

- 空：展示 3 个入口和一个轻量说明。
- 拖拽中：边框变菠萝金，背景轻微变亮。
- 解析中：显示文件名、进度条和当前步骤。
- 成功：展示识别到的经历数量。
- 失败：明确原因和下一步，例如 `PDF 无法提取文本，请上传 DOCX 或粘贴文本。`

### 4. 证据库

证据库是产品可信度核心，视觉要比普通列表更强。

卡片字段：

- 时间
- 公司/项目
- 角色
- 关键动作
- 结果数据
- 来源标签
- 证据状态

证据状态标签：

- `Verified` / `已确认`：绿色
- `Needs detail` / `需补充`：菠萝金
- `Unsupported` / `无依据`：珊瑚红
- `From JD` / `来自 JD`：深蓝

每条证据左侧可以使用微型菠萝菱格图标，表示它已经进入 evidence graph。

### 5. JD 分析

JD 模块要克制，不要做夸张仪表盘。

必须展示：

- 岗位名称/公司/语言
- 关键词组
- 硬技能
- 软技能
- 与现有经历的匹配点
- 缺口和追问建议

匹配分不要叫 `ATS score`，推荐叫：

- 中文：`岗位贴合度`
- 英文：`Role fit`

分数旁边必须有解释：`用于排序和追问，不代表通过率。`

### 6. AI 追问

AI 追问区要像一个专业 interviewer，不像聊天机器人玩具。

每个问题必须包含：

- 问题正文
- 为什么问
- 期望回答格式
- 快捷回复 chips
- 跳过/无法提供入口

示例：

```text
你提到“负责用户增长”。请补充：
1. 目标指标是什么？
2. 你直接做了哪 3 个动作？
3. 最终数据变化是多少？
```

视觉：

- 问题正文使用浅绿色背景。
- “为什么问”使用细边框说明块。
- chips 使用短标签，不要长句。

### 7. 简历预览

简历预览应该像真实可导出的文档，不像普通网页卡片。

布局：

- 白色纸张背景
- 细边框
- 内边距桌面 `32px`
- 宽高比例接近 A4，可滚动
- 姓名和职位有明确文档层级

每条 bullet 右侧或下方必须能看到来源状态：

- 来源 ID，例如 `EXP-03`
- 证据状态
- 是否需要确认

不要把来源追踪藏到 hover 里；移动端没有 hover。

### 8. 导出区

V1 优先：

- Copy
- Markdown
- PDF

DOCX 可以作为 V1.1。除非产品 owner 明确要求，不要把 DOCX 做成首要按钮。

导出前必须有校验：

- 是否存在 unsupported claim
- 是否还有必填确认
- 是否选择了目标语言

## 组件规范

### 按钮

按钮分三种：

- Primary：深墨绿底，象牙白字，用于下一步主操作。
- Secondary：纸张底，深墨绿边框，用于上传、复制、切换。
- Danger/Block：珊瑚红边框或底色，用于 unsupported claim。

按钮必须有：

- hover
- active
- focus-visible
- disabled
- loading

Focus 样式：

```css
button:focus-visible,
a:focus-visible,
input:focus-visible,
textarea:focus-visible {
  outline: 3px solid rgba(240, 184, 58, 0.55);
  outline-offset: 2px;
}
```

### 输入框

- 文本输入和 textarea 使用 `6px` 圆角。
- 边框默认 `rgba(18, 36, 33, 0.18)`。
- focus 时边框变 `--brand-gold-deep`。
- 错误状态显示红色边框 + 明确错误文案。
- textarea 右下角显示字数或解析状态。

### 标签

标签要服务信息识别，不要装饰。

推荐标签类型：

- 文件格式：`PDF`、`DOCX`
- 来源：`Profile`、`JD`、`Answer`
- 状态：`Verified`、`Needs detail`、`Unsupported`
- 语言：`ZH`、`EN`

### 图标

如果使用 icon library，优先使用 lucide icon。不要手写通用图标，Logo 除外。

图标按钮必须有 tooltip 或 `aria-label`。工具类操作优先用图标，例如复制、下载、删除、展开、收起。

## 动效

动效必须低调、可信、服务反馈。

允许：

- 页面进入时面板 `opacity + translateY(8px)`，总时长 `180ms - 240ms`。
- 证据节点被确认时轻微点亮。
- 上传进度条线性推进。
- Logo 大尺寸加载时内部节点依次亮起。

禁止：

- 大面积弹跳。
- 无限旋转 Logo。
- 背景粒子雨。
- 让简历预览漂浮摇晃。

## 必须考虑的 UI 状态

每个新流程都要考虑：

- 空状态
- 加载状态
- 成功状态
- 校验错误
- 上传失败
- AI 因证据不足无法生成
- unsupported claim 被阻断
- 移动端布局
- 英文文案变长后的布局
- 键盘 focus
- 低网速或接口超时

## 页面文案原则

文案要像专业产品，不要像营销广告。

推荐：

- `证据不足，先补充这段经历。`
- `这条 bullet 来自 EXP-03 和 JD-02。`
- `岗位贴合度用于发现缺口，不代表通过率。`
- `复制已确认内容`

避免：

- `一键打造完美简历`
- `保证 ATS 通过`
- `秒杀 HR`
- `AI 自动帮你搞定所有求职材料`

## 验收标准

视觉验收：

- 1440px 桌面宽度下，首屏能同时看见流程入口、JD 匹配、AI 追问和简历预览。
- 390px 移动端宽度下没有横向溢出。
- 中文和英文界面文案都不能被裁切。
- 面板之间的对齐、间距、边框粗细统一。
- Logo 在 `24px`、`40px`、`72px` 三种尺寸下都清晰。
- 页面不能读成普通紫色 SaaS 模板，也不能读成果汁/水果品牌。

交互验收：

- 按钮动作明确，并且有可见的 focus 状态。
- 上传、解析、AI 生成、证据不足、导出失败都有明确状态。
- 不支持的内容不能进入最终简历预览，必须转成追问或标红阻断。
- 移动端所有主流程都可以单手完成。

产品验收：

- 首屏能表达产品工作流程，但不是纯营销页。
- 每条生成内容都能看到来源依据。
- 不承诺不现实的自动化结果。
- 用户能明确知道下一步该做什么。
