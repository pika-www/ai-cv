# V1 严格验收标准

## 发布原则

如果下面的 P0 条目有任何一条不过，不能把 V1 给朋友试用。

如果 P1 条目不过，可以内部自用，但不算完成。

## P0 发布阻断项

### 核心闭环

- 用户能粘贴简历文本。
- 用户能上传旧简历文件。
- 上传失败时，用户能改用粘贴文本继续。
- 用户能点击分析按钮。
- 系统能在右侧返回可理解的优化建议。
- 用户能在左侧直接编辑简历。
- 用户能输入新增经历。
- AI 能把新增经历放入简历合适位置。
- 用户能重复分析和编辑。
- 用户能导出 PDF。
- 导出的 PDF 和预览在版式上保持一致。

### 编辑真实性

- AI 不能编造公司、时间、职位、学历、工具、指标或结果。
- 没有证据的内容不能直接进入可导出结果。
- 需要确认的内容必须明确标记。
- 被阻断的内容不能混进导出结果。
- 用户删掉的内容不能在下一轮无提示地回来。
- 新增经历只能基于用户输入生成，不能扩写成用户没有提供过的事实。

### AI Key 与安全

- AI key 只能在后端环境变量里。
- 前端源码、构建产物、浏览器 storage、网络响应里都不能出现 AI key。
- `.env` 文件不能提交。
- 日志不能记录完整简历、手机号、邮箱、地址、API key 或原始 provider response。
- provider 调用必须有超时。
- provider 失败时必须显示可恢复错误。
- 朋友试用版本必须至少有简单访问控制，不能无保护公网开放。

### UI 与体验

- 主界面必须是简历编辑器，不是纯文本审计报告。
- 右侧建议不能抢走主视图。
- 上传入口、粘贴入口、新增经历入口都必须容易找到。
- 390px 宽度不能出现横向滚动。
- 中文长文案和英文长文案都不能裁切。
- 可交互元素必须有可见 focus 状态。
- 用户返回上一步时，已填内容不能丢。

### 导出

- 导出前必须重新校验当前草稿状态。
- blocked 或未确认的内容不能进入最终 PDF。
- 导出失败时，当前编辑内容不能丢失。
- PDF 文件名应包含日期和简历名称线索。

## P1 质量要求

### 可用性

- 新用户不需要开发者解释，也能完成一次完整编辑。
- 用户能看懂 AI 为什么做了某个改动。
- 用户能在 10 到 20 分钟内把一份现有简历改到可导出状态。

### 预览一致性

- 预览和 PDF 使用同一模板。
- 页面分页、字体、间距、图片和分栏在预览和导出中保持一致。
- 不允许导出后出现明显排版漂移。

### AI 输出质量

- 修改结果要自然，不像机器拼接。
- 中英文文案不能直译腔。
- 没有数字时不要硬编数字。
- 用户提供估算时要标记为估算或待确认。

### 编辑模型

- 每个编辑项都有稳定的 ID。
- 每轮 AI 建议都能被回滚。
- 至少支持一个稳定草稿版本。
- 建议能映射到具体文本区域。

## P2 可后置但建议尽快补齐

- DOCX 导出。
- 简历版本历史面板。
- 目标岗位/JD 对比。
- 更精细的模板库。
- 用户删除当前会话数据。
- 细粒度操作日志。

## 手动验收用例

### Case 1: 导入并预览

输入：

- 一份真实旧简历。

必须通过：

- 页面立即出现可编辑文本。
- 用户能看见建议区和编辑区。

### Case 1B: 上传旧简历

输入：

- 一份 `.pdf`、`.docx` 或 `.txt` 旧简历。

必须通过：

- 系统能解析出可编辑文本，或在解析失败时提供粘贴文本替代路径。
- 用户可以点击分析按钮。
- 右侧能展示优化建议。

### Case 2: add / modify / delete

输入：

- “把这段改得更简洁。”
- “帮我突出 Rust 后端经历。”
- “删除重复内容。”

必须通过：

- AI 输出具体建议。
- 用户能手动应用或忽略。
- 预览随之更新。

### Case 2B: 新增经历自动插入

输入：

- 当前简历已有工作经历和项目经历。
- 用户输入：“新增经历：我做了 Redis 缓存优化，把接口平均响应时间从 800ms 降到 300ms。”

必须通过：

- AI 判断应该放入工作经历或项目经历。
- 系统展示插入后的简历内容。
- 用户可以接受、编辑或撤销。
- 插入内容不能编造额外公司、时间、工具或结果。

### Case 3: 证据不足

输入：

- 用户没有提供某项指标。
- 但建议想写进简历。

必须通过：

- 系统阻断该内容或提示用户确认。
- 不能直接进入最终 PDF。

### Case 4: 导出一致性

输入：

- 完成一轮编辑后导出 PDF。

必须通过：

- 导出文件与预览一致。
- 不出现重新排版导致的明显错位。

### Case 5: AI 失败

输入：

- 模拟 provider timeout 或无效 key。

必须通过：

- 页面显示可恢复错误。
- 用户已输入内容不丢失。
- 用户可以重试。

### Case 6: 移动端

环境：

- 390px 宽度。

必须通过：

- 无横向滚动。
- 预览可查看。
- AI 指令可输入。
- 导出入口可达。
